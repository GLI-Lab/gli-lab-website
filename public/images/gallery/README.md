# Gallery 폴더 사용법

이 폴더는 웹사이트의 갤러리 페이지에서 사용할 이미지들을 저장하는 곳입니다.

## 폴더 구조

각 프로젝트나 갤러리 아이템은 개별 폴더로 관리됩니다:

```
gallery/
├── project-1/
│   ├── config.yaml
│   ├── image1.jpg
│   ├── image2.png
│   └── ...
├── project-2/
│   ├── config.yaml
│   ├── photo1.jpg
│   └── ...
└── README.md
```

## 새 갤러리 아이템 추가하기

1. **폴더 생성**: 프로젝트 ID로 사용할 폴더명으로 새 폴더를 생성합니다.
   - 폴더명은 URL에 사용되므로 영문, 숫자, 하이픈(-), 언더스코어(_)만 사용하세요.
   - 예: `ai-research-2024`, `web-project-1`

2. **이미지 추가**: 폴더 안에 이미지 파일들을 추가합니다.
   - 지원 형식: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.svg`
   - 첫 번째 이미지가 썸네일로 사용됩니다.

3. **설정 파일 생성** (선택사항): `config.yaml` 파일을 생성하여 메타데이터를 추가합니다.

## config.yaml 형식

```yaml
title: "프로젝트 제목"
description: |
  프로젝트에 대한 상세 설명
  
  여러 줄로 작성 가능합니다.
date: "2024-03-15"
```

또는 `\n`을 사용하여 한 줄로 작성할 수도 있습니다:

```yaml
title: "프로젝트 제목"
description: "프로젝트에 대한 상세 설명\n\n여러 줄로 작성 가능합니다."
date: "2024-03-15"
```

### 필드 설명

- `title` (선택사항): 갤러리에 표시될 제목. 없으면 폴더명이 사용됩니다.
- `description` (선택사항): 상세 설명. Details 버튼을 눌렀을 때 모달에 표시됩니다.
- `date` (선택사항): 프로젝트 날짜. YYYY-MM-DD 형식으로 입력하세요.

## 주의사항

- 이미지 파일 크기는 적절히 최적화해주세요 (권장: 1-2MB 이하)
- 폴더명은 변경하지 마세요 (URL이 변경됩니다)
- config.yaml 파일의 YAML 형식이 올바른지 확인해주세요
